# react井字棋学习总结
这个项目将一个井字棋逐渐的拆解成三个组件 =》负责每个方块的square，棋盘的board，以及其他游戏功能的game。

在此之间穿插进了一些JSX的语法，包括：
- 每个JSX需要用一个标签包裹（ 可以是空标签<></> )

- class为避免关键字改为className，点击事件为onClick

- 使用条件循环时需要添加key

在点击事件中有使用props的方式把点击函数传递到对应的组件中，但是不可以直接使用onClikc={propsEvent()} 的形式，会导致渲染时直接执行。onClick={propsEvent}可以

也有两点比较重要的React哲学：
- 状态提升：整个例子中变量提升发生了很多次，squares从board到game
状态提升的好处时：统一了数据源，避免数据混乱，简化数据逻辑，
- 不变性赋值：在这个游戏中因为又时间回溯的需求，所以每次都会把上一次的状态浅拷贝一份才进行赋值，这样避免影响到上一次的状态。

react渲染规则是：
- 组件调用自身setState 一定会重新执行render
- 父组件执行render -- 所有的子组件都会render
- 与props、是否可变更新无关
